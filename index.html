Javascript Testing

Why Should I Test?

  • Manually testing in the browser is time intensive
  • Confidently refactor your code without fear of regression bugs
  • Gives clear direction of what needs to happen next
  • Exposes code smells

Types of Tests

  • Specs (specifications) is also used to describe tests
  • Unit Tests - testing individual units of code in isolation
  • Integration Tests - testing multiple components of code as a group
  • Feature Tests (aka E2E - End-To-End) - a high-level Integration Test. Usually simulates what a user experiences in the browser

Right Ratio when Testing

E2E tests = more setup and take longer to run. Usually only cover a single "Happy Path"

Unit tests = run faster. Usually cover all edge cases. (ex - all combinations to check a password's validity)

Testing Tools

  • Karma - Node app that starts a web server to load your code and run your tests
  • Jasmine - framework that provides functions to help with structuring your tests and also making assertions
  • Chai - offers various ways of checking things in test cases
  • Sinon - standalone test spies, stubs and mocks (allows to better isolate your code)

Test Driven Development

There are different philosophies of when to write your tests and there is a lot of love and push back for TDD.

TDD in Action

Red, Green, Refactor. Write a failing test first, then only write enough code for your test to pass and turn green. Refactor your code and make sure the test stays green.

Continouous Integration

Automated build process that uses an application's test suite to catch errors before deploying.
